Android crypto mining

With the recent "crypto boom" I started getting into it cryptocurrencies. I purchased some cryptocurrencies (Cardano & Algorand mainly) and I started wondering if I could mine with my Raspberry Pi. Looked it up and in ten minutes I was mining with my Raspberry Pi. Great! Well that was simple!

The hashrate was really low, being about 100h/s. (Hashrate is a simply but the speed of mining. It is measured in units of hash/second, meaning how many calculations per second can be performed.) And 100h/s is really slow compared to a quite low end modern CPU, for example the Ryzen 5 2600 CPU. That one can do about 4000h/s and even more.

But then I tought, "If I can mine crypto with Raspberry Pi, surely I could with a Android device?" And then I started researching how to get a terminal in Android. Thankfully in the Google Play Store there was already a terminal app called Termux. Termux seems to be the only terminal emulator that has succesfully made a package manager within the app. With a package manager we can add custom packages and that is necessary for this to work.

In this blog post I will be answering three questions:
- Why?
- How?
- Should I?

Why?

The main reason for this is to learn.
You wont be going to make money from mining with your Android device. (Maybe in the long run but more on that later)
People have also told me that they mine with their Android devices because it supports the blockchain, not greatly but it's better than nothing.



How?

Installation, Build & Run instructions
Please run the commands without the ' marks.


1. Download & Install Termux app from the Google Play Store for free.
2. Open Termux and update the dependencies: 'apt-get update && apt-get upgrade'
3. Install git, build-essential & cmake: '​pkg install git build-essential cmake -y'​
4. Get XMRig from git: 'git clone https://github.com/xmrig/xmrig.git'
5. Make build dir and navigate to the directory: 'mkdir xmrig/build && cd xmrig/build'
6. Build XMRig with cmake: 'cmake .. -DWITH_HWLOC=OFF && make -j$(nproc)'
7.1 Run XMRig with a command or create a config file. You can create a config file from https://xmrig.com/wizard.
7.2 Running XMRig with command: './xmrig -o {POOL_ADDRESS} -a rx -k -u {USERNAME} -p x'
Please note that this is just an example. Please follow the pools instructions how to point your miner to the right place.

And with that, you are all set up and can start mining!

RECOMMENDED:
Manually set "acquire wakelock" by pulling down on Termux app notification. This tells Android to not mess with the Termux app. Thermal throttling will quickly occur to protect itself, no additional action is actually needed.

KNOWN ISSUES:
Wakelock may not be adequate to protect against the Android OS overriding and killing the XMRig thread within Termux.
You can try to turn the battery optimization off for the app from the settings! This at least worked for me, and I have been running the phone with the screen closed for more hashes. But I think this option is only available for the newer devices.



Should I?

My toughts, Results & Conclusion

Eh, maybe if you have a spare phone that you won't be using anymore? The device needs to be plugged in 24/7 of course, and that will degrade the battery with the combination of the heat of the CPU. Also, the long time returns are very small when compared to how much the phone has cost.

With that said, this setup uses very little power in return with a "pretty good" hashrate. I recommend people to test this for fun, at least if you have some devices just lying around and collecting dust. But be aware that the hashrate will be different on older and newer devices! I tried using this with an old tablet and got an amazing 50 hashrate on that. So the device should have a good amount of RAM because my old tablet only has 1GB!

Also the CPU's cache plays an important part because with more cache XMRig can operate more threads.
More threads == bigger hashrate


Results
Currently, on my Pocophone F1 that has 6GB RAM & is running on Snapdragon 845. I'm getting a steady 450 hashes and have been running for over two days straight.

The phone is drawing ~3.8W from the outlet and with some calculations profit per day is about $0.02375 and yearly profit is about $8.67.

After running the phone for 24 hours the phone used about 0,101195kWh of electricity so in total it cost $0,00121434 to run for a day. (0,101195kWh * $0,12 = $0,0121434)
So with my electricity plan it costs nearly nothing to run it all day, all night with daily profits of $0.02 .


Conclusion
A bit of an anticlimactic results isn't it? I didn't have my expectations very high with this project but maybe I expected a bit different results.

At least in my opinion it isn't worth it / wise to mine cryptocurrencies with your Android device.

But as an learning experience it was really fun and worth it. I learned a lot about cryptocurrencies and how to mine them when researching for this blog post, and when figuring out how to start mining on Android device.
